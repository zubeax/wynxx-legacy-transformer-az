generate a REXX script that reads input files with account numbers, account holder names
and payment purpose and caches them in stem variables.
the script then generates payment transactions by generating random amounts between
1 EUR and 1000 EUR and selecting random creditor and debtor details from the stem
variable cache. the execution date is picked randomly between the minimum and maximum
dates passed as parameters. the number of transactions to generate is also passed 
as a parameter.
write the generated transactions to a fixed block output file.
generate jcl to execute the script in an IRXJCL step that reads the script
from an instream dataset.
create instream datasets for account numbers and account holder details 
with 50 randomly generated records.
the output dataset is sorted by execution date and creditor account in an ICETOOL step.
the output from the sort step should be persisted in a fixed block dataset.

Generate a readme.md file documenting all generation steps and the usage of the job.
