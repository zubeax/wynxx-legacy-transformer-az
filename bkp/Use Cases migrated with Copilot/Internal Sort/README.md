## Assumptions & Notes
- **Character set**: If your data is **EBCDIC**, convert to **ASCII/ISO-8859-1** before running (e.g., using iconv). The program uses `ISO-8859-1` to avoid multi-byte behavior that would break fixed-width slicing.
- **Collating sequence**: COBOL `PIC X` comparisons are alphanumeric. Java lexicographic compare over `ISO-8859-1` strings matches this intent on ASCII/Latin inputs.
- **Record length**: Strictly 80 characters per record in and out.
- **Memory**: This implementation sorts **in memory**. For very large datasets that cannot fit in memory, consider an **external merge sort** approach (chunking to temp files, sorting chunks, then multi-way merging).
